# HTML Cheatsheet
Quick references you’ll actually use.

──────────────────────────────────────────────────────────────────────────────
HTML5 BOILERPLATE (minimal)
──────────────────────────────────────────────────────────────────────────────
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>My First Webpage</title>
  </head>
  <body>
  </body>
</html>

Why it matters:
• <!DOCTYPE html> → standards mode (modern HTML)
• lang="en" → accessibility & SEO
• meta viewport → proper scaling on mobile
• title → tab text / bookmarking

──────────────────────────────────────────────────────────────────────────────
TEXT & EMPHASIS
──────────────────────────────────────────────────────────────────────────────
<strong>…</strong>  → strong importance (often renders bold)
<em>…</em>          → emphasis (often renders italic)

Tip: These are semantic, not just “bold/italic.” Screen readers announce them.

──────────────────────────────────────────────────────────────────────────────
LISTS
──────────────────────────────────────────────────────────────────────────────
Unordered:
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>

Ordered:
<ol>
  <li>Step 1</li>
  <li>Step 2</li>
</ol>

──────────────────────────────────────────────────────────────────────────────
LINKS (ANCHORS)
──────────────────────────────────────────────────────────────────────────────
<a href="https://www.theodinproject.com/about"
   target="_blank"
   rel="noopener noreferrer">About The Odin Project</a>

Notes:
• target="_blank" → open in new tab
• rel="noopener" → new tab cannot control the opener (security)
• rel="noreferrer" → also hides the referrer (privacy) and implies noopener
• Use same-tab links for your own pages (internal navigation) by default.

──────────────────────────────────────────────────────────────────────────────
PATHS: ABSOLUTE vs RELATIVE
──────────────────────────────────────────────────────────────────────────────
Absolute URL (web):
  https://example.com/assets/logo.png

Relative (filesystem inside your project):
  ./pages/about.html   → current folder (.)
  ../images/dog.jpg    → parent folder (..)

Tip: Relative paths depend on the current file’s location.

──────────────────────────────────────────────────────────────────────────────
IMAGES
──────────────────────────────────────────────────────────────────────────────
Absolute URL example:
<img src="https://www.theodinproject.com/mstile-310x310.png"
     alt="The Odin Project logo"
     width="310" height="310" loading="lazy">

Relative path example (from /pages/file.html):
<img src="../images/dog.jpg"
     alt="Pug looking at the camera"
     width="310" height="310" loading="lazy">

Notes:
• Always include meaningful alt text (accessibility). If decorative, alt="".
• width/height help reserve layout space. Keep aspect ratio correct.
• Use CSS to control display size; attributes define intrinsic size.
• loading="lazy" defers off-screen images for performance.
